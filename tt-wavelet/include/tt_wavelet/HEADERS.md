# compile-time wavelet lifting schemes

## File layout

```
include/tt_wavelet/
├── lifting_step.hpp        - StepType enum + LiftingStep<Type, N> template
├── lifting_scheme.hpp      - LiftingScheme<Steps...>, WaveletScheme<Id>, SchemeInfo
├── wavelet_registry.hpp    - GENERATED: all 106 includes + find_scheme() + WaveletScheme<Id>
└── wavelets/
    ├── haar.hpp            - GENERATED: haar_scheme + scheme_traits<haar_tag>
    ├── db4.hpp             - GENERATED
    └── ...                 - 106 files total
```

`lifting_step.hpp` and `lifting_scheme.hpp` are hand written. Everything in `wavelets/` and `wavelet_registry.hpp` is generated by `scripts/add_lifting_scheme.py`.

## idea

### host

The host looks up the wavelet by name at runtime, gets its ID, and passes it to the JIT compiler as a single preprocessor define:

```cpp
#include <tt_wavelet/wavelet_registry.hpp>
#include <string>

const ttwv::SchemeInfo* info = ttwv::find_scheme("db4");
// info->id        = 64
// info->tap_size  = 8
// info->delay_even = 2
// info->delay_odd  = 2

tt::tt_metal::CreateKernel(
    program,
    "kernels/wavelet_sfpu.cpp",
    core,
    tt::tt_metal::ComputeConfig{
        .defines = {{"WAVELET_ID", std::to_string(info->id)}} // <- here
    }
);
```

### kernel

The kernel includes the full registry (`wavelet_registry.hpp`) and selects the active scheme via template dispatch on `WAVELET_ID`:

```cpp
#include <tt_wavelet/wavelet_registry.hpp>

// WAVELET_ID is injected by the JIT compiler as -DWAVELET_ID=64
constexpr auto& scheme = ttwv::WaveletScheme<WAVELET_ID>::scheme;

// everything is available at compile time

scheme.for_each_step([](const auto& step) {
    using S = std::remove_cvref_t<decltype(step)>;

    if constexpr (S::type == ttwv::StepType::Predict) {
    } else if constexpr (S::type == ttwv::StepType::Update) {
        // ...
    } else if constexpr (S::type == ttwv::StepType::ScaleEven) {
        // ...
    } else if constexpr (S::type == ttwv::StepType::ScaleOdd) {
        // ...
    } else if constexpr (S::type == ttwv::StepType::Swap) {
        // ...
    }
});
```

## types

### `StepType`

```cpp
enum class StepType : uint8_t {
    Predict,
    Update,
    ScaleEven,
    ScaleOdd,
    Swap,
};
```

### `LiftingStep<Type, N>`

A single lifting step.

```cpp
static constexpr StepType    type;
static constexpr std::size_t n_coeffs;
std::array<float, N>         coefficients;
int                          shift;
```

Aliases: `PredictStep<N>`, `UpdateStep<N>`, `ScaleEvenStep<N>`, `ScaleOddStep<N>`, `SwapStep`.

### `LiftingScheme<Steps...>`

Heterogeneous tuple of steps. Key members:

```cpp
int    tap_size;
int    delay_even;
int    delay_odd;
static constexpr std::size_t num_steps;

scheme.step<I>()
scheme.for_each_step(v)
scheme.for_each_step_indexed(v)
```

### `WaveletScheme<Id>`

Maps an integer ID to a concrete scheme.

```cpp
WaveletScheme<71>::scheme     // haar
WaveletScheme<64>::scheme     // db4
WaveletScheme<WAVELET_ID>::scheme  // kernel usage
```

### `SchemeInfo`

Runtime POD returned by `find_scheme()`. Host-only - never use in kernel code.

```cpp
struct SchemeInfo {
    const char* name;
    int id;
    int tap_size;
    int delay_even;
    int delay_odd;
    int num_steps;
};
```

### `scheme_traits<Tag>`

Compile-time access by tag type (made only for host-side nice looking code):

```cpp
using T = ttwv::scheme_traits<ttwv::db4_tag>;
static_assert(T::id == 64);
static_assert(T::tap_size == 8);
constexpr auto& s = T::scheme;
```
